# 移动端适配需求文档 (Requirements)

## 1. 背景与目标

为了扩大 X-Puzzle-Kit 的使用场景，本项目计划在现有 Chrome 扩展的基础上，提供对 Android 和 iOS 移动端的支持。目标是复用 90% 以上的现有业务代码，实现“双端发布，一套维护”。

## 2. 核心用户需求

- **Android 用户**：能够通过 Twitter 系统分享菜单直接导入图片到拼图工具。
- **iOS 用户**：通过快捷指令一键下载推文图片并导入工具，解决系统级的跨域与分享限制。
- **通用需求**：工具需支持“添加到桌面”(PWA)，提供接近原生 App 的全屏交互体验。

## 3. 平台限制分析

| 平台        | 获取方式       | 技术约束                    | 解决方案                      |
| :---------- | :------------- | :-------------------------- | :---------------------------- |
| **Android** | 系统分享菜单   | 需支持 Web Share Target API | PWA `manifest.json` 配置      |
| **iOS**     | 快捷指令跳转   | 不支持 Web Share Target     | 快捷指令抓取图片 + 剪贴板桥接 |
| **Web**     | PWA / 域名访问 | 无 Chrome Extension API     | 建立 Platform Adapter 抽象层  |

## 4. 功能范围

- [ ] 核心拼图 (Stitch) 逻辑在移动浏览器运行。
- [ ] 核心切图 (Split) 逻辑在移动浏览器运行。
- [ ] 响应式 UI：适配 375px~430px 宽度的竖屏显示。
- [ ] 本地设置持久化 (localStorage)。
